# 题材映射与标的池字典 v0.1

## 1. 目标

本字典是 Versefina 1.6 P0 最关键的基础资产之一。

它用于回答：

- 先影响哪个商品
- 再影响哪个产业链环节
- 再影响哪个行业
- 再影响哪些股票
- 龙头、跟风、补涨谁可能先动

---

## 2. 字典结构

统一映射结构：

```text
商品
→ 产业链环节
→ 行业 / 题材
→ A 股标的池
→ 风格标签
```

---

## 3. 建议字段

### `commodity`

事件直接冲击的商品或原料。

### `chain_stage`

产业链位置，例如：

- 上游
- 中游
- 下游

### `sector`

行业或题材标签。

### `symbols`

关联标的池。

### `style_tags`

建议至少支持：

- `龙头`
- `高弹性`
- `机构票`
- `情绪票`
- `补涨候选`

---

## 4. 第一版使用原则

- 先做小而稳的人工维护版本
- 不追求覆盖全市场
- 只覆盖首发事件白名单所需商品和链条
- 每个商品至少有一条完整映射链

---

## 5. 示例

### 样例一

```text
某化工原料
→ 上游化工
→ 化工 / 新材料
→ [标的A, 标的B, 标的C]
→ [龙头, 高弹性, 情绪票]
```

### 样例二

```text
某金属
→ 上游有色
→ 有色金属
→ [标的D, 标的E]
→ [机构票, 龙头]
```

---

## 6. 使用要求

- Event Structuring 输出必须能够引用本字典
- Scenario Engine 需要依赖该字典生成 `first_movers / secondary_movers`
- Lightweight Simulation 需要依赖该字典生成参与者动作对象

---

## 7. 维护规则

- 本字典可以先人工维护
- 每次扩展事件类型时优先扩展字典，而不是先扩展 prompt
- 同一商品的映射链必须可追溯、可解释、可修改
