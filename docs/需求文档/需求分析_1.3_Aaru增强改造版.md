# Agent-native 金融世界平台 v1.2（Aaru 增强改造版）

## 文档说明

本文件不是重写新系统，而是基于现有两份方案做增量增强：

- `需求分析_1.1.md`：**交割单驱动的行为镜像 Agent 平台**
- `需求分析_1.2.md`：**事件驱动的金融反身性引擎**

本改造版的目标是：

**在不推翻你现有闭环的前提下，引入 Aaru 范式中最有价值的三层能力：**

1. **Synthetic Market Participants**：不是只做功能 Agent，而是补一层“市场参与者画像/风格族群”
2. **Behavioral Architecture**：每个金融 Agent 不只是 prompt，而要有身份、偏好、约束、反应规则
3. **Distribution Calibration**：不只做单事件判断，而要按事件类型、参与者群体、剧本分布做校准

---

# 一、现有方案复盘：哪些不动，哪些增强

## 1.1 现有 v1.1 的核心价值（保留）

你现有 v1.1 的核心闭环已经成立：

```text
交割单上传
→ 解析与标准化
→ 行为特征提取
→ 交易画像生成
→ 策略模板匹配
→ 受限策略壳生成
→ 行为镜像 Agent 创建
→ 统一日线世界运行
→ 留出集验证与乱交易检测
→ Agent 分级
→ 只读观察台与公开页展示
```

这条链最重要的价值是：

- 输入成本低（只要交割单）
- 先做“行为保真”而不是“神级赚钱”
- 模板化策略壳抑制 LLM 自由发挥
- 统一世界运行 + 验证分级 + 只读围观

**这一整条主线保留，不动。**

## 1.2 现有 v1.2 的核心价值（保留）

你现有 v1.2 已经把“新闻总结”升级成“反身性剧本系统”：

```text
事件
→ 8 类 Agent 形成观点
→ 抽取共识/分歧结构
→ Belief Graph
→ 资金状态迁移
→ 三剧本
→ T+1/T+3 验证与复盘
```

这条链最重要的价值是：

- 不直接预测价格，而预测“预期—行为—价格—再预期”
- 引入共识/分歧、拥挤度、脆弱性等中间状态
- 输出剧本，而不是单点结论
- 支持后验验证和 Agent 打分

**这一条主线也保留，不动。**

## 1.3 Aaru 范式真正值得借鉴的部分

Aaru 真正值得借鉴的，不是“并发几千个 Agent”，而是：

- 先造 **synthetic population**，再看群体反应
- 给 Agent 加上 **behavioral architecture**，而不只是聊天能力
- 用 **population/distribution-level calibration** 做验证，而不是迷信单案例

迁移到你的金融系统里，就是：

- 不是只做“8 个固定功能 Agent”
- 而是在它们上面再补一层 **Synthetic Market Participants / 市场参与者仿真层**

# 二、改造总原则

## 2.1 不推翻现有系统，只补中间层

改造原则：

- **不动 v1.1 的行为镜像主线**
- **不动 v1.2 的 Belief Graph + Scenario Engine 主线**
- **新增一层：Market Participant Simulation Layer**

也就是把原来的两条能力链接起来：

```text
交割单 / 历史行为
→ 行为镜像与风格画像
→ Market Participant Archetypes（新增）
→ 事件输入
→ Agent 社会分歧
→ Belief Graph
→ Scenario Engine
→ Outcome Tracker
```

## 2.2 从“功能 Agent”升级为“参与者 Agent”

原来的 8 个 Agent 更像功能模块。

增强后，8 个 Agent 仍然保留，但每个 Agent 不再只是功能角色，而要绑定：

- 市场参与者身份（例如：游资、公募、政策研究员、量化、风控）
- 行为风格（追涨/低吸/确认后跟随/防守）
- 风险预算
- 触发/撤退规则

这样它们输出的就不是“分析文本”，而是“带交易后果的市场信念”。

## 2.3 从“单事件正确性”升级为“分布校准”

增强后不只验证某次事件猜得准不准，而要验证：

- 同类事件里，哪类 Agent 在什么环境下更可靠
- 哪类分歧结构更容易演化为主线
- 哪类拥挤结构更容易崩
- 哪类事件更适合哪类市场参与者

---

# 三、新增核心层：Market Participant Simulation Layer

这是本次增强最核心的部分。

## 3.1 定义

新增一个“市场参与者仿真层”，位于：

- v1.1 的 **交易画像/行为镜像层** 与
- v1.2 的 **Agent 社会/Belief Graph 层**

之间。

它的作用是：

**把交易者历史行为抽象成“市场参与者原型”，再让这些参与者在新事件下形成分歧与共识。**

## 3.2 为什么要加这一层

否则你当前系统存在两个断层：

### 断层 A：
交割单生成的行为镜像 Agent，偏向“这个人过去怎么交易”，但未必能自然过渡到“在新事件情境下，这类参与者会如何反应”。

### 断层 B：
反身性引擎里的 8 个 Agent，偏向“功能角色分析器”，但没有足够强的市场参与者异质性。

Market Participant Simulation Layer 的作用就是补这两个断层。

## 3.3 这一层输出什么

输出不是自然语言，而是：

### MarketParticipantProfile
- participant_id
- archetype_name
- participant_family（游资 / 公募 / 量化 / 政策研究 / 风控 / 情绪传播）
- style_embedding
- risk_budget_profile
- holding_preference
- reaction_rules
- invalidation_sensitivity
- authority_weight

### 例子

#### Archetype: Fast Momentum 游资型
- 风格：追涨、偏高 Beta、重视板块合力
- 增强条件：龙头封板、成交放大、题材一致性增强
- 撤退条件：高开低走、龙头炸板、分歧急剧扩大

#### Archetype: Slow Confirmation 公募型
- 风格：中线确认、偏基本面、偏行业龙头
- 增强条件：政策覆盖范围明晰、产业逻辑兑现、估值可接受
- 撤退条件：落地慢于预期、业绩弹性不足、机构不跟

# 四、如何增强 v1.1：行为镜像 Agent 平台

## 4.1 当前优势

v1.1 已经能做：

- 解析交割单
- 提取行为特征
- 生成交易画像
- 匹配策略模板
- 生成受限策略壳
- 做验证和分级

## 4.2 Aaru 式增强方向

### 增强点 1：从交易画像到 MarketStyleEmbedding

现状：
交易画像主要描述“像谁”。

增强后：
新增 `market_style_embedding`，把交易行为向量化为可用于参与者仿真的风格坐标。

建议新增字段：

- holding_period_distribution
- add_reduce_pattern
- event_dependency_score
- momentum_preference_score
- mean_reversion_score
- concentration_score
- drawdown_tolerance
- follow_through_requirement
- reaction_latency_profile

### 增强点 2：从单一镜像 Agent 到镜像 Agent + 风格母体

现状：
上传交割单后直接生成行为镜像 Agent。

增强后：
分两步：

1. 生成 **行为镜像 Agent**（保留）
2. 同时抽象出一个 **风格母体 / Archetype Seed**

这个风格母体不直接参与公开运行，而是作为反身性引擎里的市场参与者模板来源。

### 增强点 3：增加“行为校准标签”

新增：
- 是否事件驱动型
- 是否情绪博弈型
- 是否趋势跟随型
- 是否确认后跟随型
- 是否逆势/均值回归型
- 是否高拥挤承受型

这会直接帮助 v1.2 的 Belief Engine 做加权。

---

# 五、如何增强 v1.2：金融反身性引擎

## 5.1 当前优势

v1.2 已经有：

- 8 个 Agent
- Belief Graph
- 资金状态迁移
- 三剧本
- Outcome Tracker

## 5.2 Aaru 式增强方向

### 增强点 1：8 个 Agent 变成“角色族群 + 风格变体”

现状：
8 个 Agent 是固定角色。

增强后：
每类 Agent 内部允许有风格变体：

例如：

#### 游资风格 Agent
- Fast Breakout 型
- High Conviction Leader 型
- Follow Trend 跟风型

#### 机构风格 Agent
- Slow Confirmation 型
- Fundamental Re-rating 型
- Defensive Allocation 型

这样你的“8 个 Agent”不再只是 8 个功能分析器，而变成：

**8 类参与者家族**。

MVP 不必全开，只需为每类保留 1 个主变体 + 可扩展字段。

### 增强点 2：Belief Graph 增加“参与者构成”字段

建议新增：

- consensus_composition
- dissent_composition
- conviction_skew
- fast_money_alignment
- slow_money_alignment
- retail_style_alignment
- institutional_style_alignment

这样你不只是知道“有共识”，还知道：

**这个共识到底是谁形成的。**

这很关键，因为金融里“游资共识”和“机构共识”不是同一种东西。

### 增强点 3：Scenario Engine 不只推演结果，还推演“共识结构如何演化”

原来：
Belief Graph → 资金状态迁移 → 三剧本

增强后：
Belief Graph → 共识结构演化 → 资金状态迁移 → 三剧本

新增中间变量：
- consensus_acceleration
- dissent_decay
- reversal_risk
- style_rotation_probability

这样系统更接近反身性，而不只是“观点聚合”。

# 六、Belief Graph 改造建议

## 6.1 原有聚合指标保留

保留原有：
- consensus_strength
- dissent_width
- confidence_mean
- confidence_dispersion
- authority_weighted_consensus
- propagation_speed
- crowding_score
- falsification_fragility

## 6.2 新增聚合指标

### 1. consensus_composition
当前共识由哪些参与者构成。

### 2. dissent_composition
当前反对声音主要来自谁。

### 3. conviction_skew
高置信度是否集中在某一类参与者上。

### 4. style_rotation_probability
如果当前逻辑继续强化，资金风格是否会切换。

### 5. reversal_risk
当前共识的反转风险。

### 6. slow_fast_divergence
快钱与慢钱之间是否发生明显分化。

这些新增字段，就是 Aaru 范式里“群体结构”的迁移版。

# 七、校准层：这是最重要的新能力

Aaru 最值得学的，是它不是只看单个 agent 像不像人，而是看群体分布是否与现实贴近。

你在金融里要迁移成：

## 7.1 校准目标

不是“每次都猜涨跌”，而是：

- 哪类 Agent 在哪类事件上更靠谱
- 哪类共识结构更容易兑现
- 哪类拥挤结构更容易失效
- 哪类参与者构成对剧本最有解释力

## 7.2 新增校准表

### `participant_segment_scores`
- segment_id
- event_type
- market_regime
- avg_hit_rate
- avg_false_alarm_rate
- avg_early_signal_score
- avg_collapse_detection_score

### `belief_pattern_scores`
- pattern_id
- event_type
- consensus_bucket
- dissent_bucket
- crowding_bucket
- hit_rate
- failure_rate

## 7.3 校准输出

系统后续要能回答：

- 在“政策催化 + 慢牛风格”环境里，谁更可信
- 在“利空扩散 + 高拥挤”环境里，哪些结构最危险
- 在“涨价链 + 快钱一致”环境里，哪些信念图谱最容易扩散

这才是你系统长出来的数据壁垒。

# 十、改造后的产品价值怎么表达

改造前，你的系统价值是：

- 上传交割单，生成行为镜像 Agent
- 输入事件，生成三剧本

改造后，你的系统价值升级为：

**不是只生成镜像 Agent，也不是只总结事件。**

而是：

**把“谁在市场里会怎样想、怎样动、怎样分歧”显性化，再把这种参与者结构喂给世界模型，生成更接近反身性的决策剧本。**

这就是 Aaru 范式真正能帮你升级的地方。

如果你现在马上开工，我建议只先做这 3 个增量点：

## 1. 给现有 8 个 Agent 增加参与者身份字段

至少新增：
- participant_family
- style_profile
- reaction_rules
- risk_budget_profile

## 2. 给 Belief Graph 增加构成字段

至少新增：
- consensus_composition

## 3. 给 Outcome Tracker 增加分群校准

不要只记录：
- 哪个 Agent 对了

还要记录：
- 哪类 Agent 在哪类事件里更靠谱

这三个点一做，你的系统就已经开始借到 Aaru 的“灵魂”了。

# 十二、一句话总结

**Aaru 对你这个金融 Agent 系统最大的启发，不是“多智能体调研”，而是：先造出一组有差异、可校准、可带行为规则的市场参与者，再让他们在事件中形成可计算的信念结构。**

你的现有方案已经有了：
- 行为镜像
- Belief Graph
- Scenario Engine
- Outcome Tracker

你真正缺的不是重写一套新系统，而是：

**在“画像”和“推演”之间，加一层“市场参与者仿真层”，再在“结果”后面加一层“分布校准层”。**

这就是这份 Aaru 增强改造版的核心。
