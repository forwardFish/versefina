# 需求分析_1.8_人口级金融参与者交易沙盘.md

## 0. 文档定位

本文件是 Versefina 在 `1.6` 事件参与者主线和 `roadmap_1_7` 动态事件沙盘主链基础上的**下一阶段详细需求文档**。

它回答三个问题：

1. **当前 Versefina 已经完成到什么程度**
2. **距离最终想要的“更接近真实金融世界”的终局效果，还差什么**
3. **下一阶段到底要做什么，开发顺序如何安排**

本版本不再停留在“8 类参与者家族级推演”，而是正式进入：

**人口级金融参与者交易沙盘阶段**

---

## 1. 当前系统完成度评估

### 1.1 已完成的核心主链

根据当前对照总结，Versefina 已经完成：

- 事件输入与结构化
- 参与者激活与参与者视角准备
- Belief Graph / 场景推演 / 市场状态
- 轻量 simulation 与逐轮快照
- replay / report / validation
- 产品页 `event-sandbox`
- Swagger 手工验证面
- agentsystem 审计面
- `finahunt -> versefina` 真实事件导入
- 事件来源 lineage 展示

这说明 Versefina 已经不再是概念 demo，而是拥有可运行、可验证、可展示、可导入真实事件的动态事件沙盘主链。

### 1.2 当前已实现但仍然不够的部分

虽然主链已打通，但距离最终效果仍有明显差距：

- 页面视觉和交互还不够像“强过程感的演化墙”
- 参与者仍然偏家族级，缺少人口级 clone
- 模拟动作虽有，但交易化程度不够
- 影响图已经有基础，但还没成为“影响如何转成买卖”的核心机制
- 市场状态机还没有和“资金行为 / 净买力 / 价格近似”深度联动
- validation 虽然存在，但缺少更系统的真实事件包、failure taxonomy、reliability scoreboard
- formal flow 治理与 authoritative roadmap 资产仍未完全收口

### 1.3 当前水平判断

当前 Versefina 处于：

**L3.5 ~ L4：可运行、可展示、可验证的事件级动态沙盘原型**

距离终局还差的，不是主线方向，而是：

1. 人口级参与者层  
2. 资金体量与交易执行层  
3. 影响到交易的转换层  
4. 市场状态更强的真实感  
5. 单事件演化墙成品感  
6. 更完整的验证与治理收口

## 2. 终局目标（本阶段视角）

### 2.1 终局一句话

**一条金融消息进来后，系统先构建事件图谱，再按市场结构激活不同参与者家族及其分身；这些分身带着不同资金体量、仓位约束、行为偏好和时间窗口，在多轮中买入、加仓、减仓、退出、传播和互相影响，最终推动市场进入点火、扩散、拥挤、脆弱或失效状态，并产出可解释的 replay / report / validation。**

### 2.2 本阶段不是做什么

本阶段不是：
- 自动下单实盘系统
- 完整撮合引擎
- 高频盘口级微结构仿真
- 1:1 复刻某个真实席位或机构

本阶段要做的是：

**分布级逼近真实市场参与者行为。**

也就是：
- 让“散户、游资、机构、量化、媒体、风控”这些类群体，在结构、资金、节奏和影响上尽量贴近真实世界。

## 3. 本阶段核心升级目标

### 3.1 从“8 类家族级”升级到“人口级”
当前系统还是 8 个 family 级核心角色。
下一阶段必须引入：

- Population Template
- Population Generator
- Clone Roster
- Clone 级交易行为

### 3.2 从“观点动作”升级到“交易动作”
当前已有动作日志，但下一阶段必须让动作交易化，至少支持：

- INIT_BUY（首次试仓）
- ADD_BUY（加仓）
- REDUCE（减仓）
- EXIT（退出）
- WATCH（观察）
- VALIDATE（验证）
- BROADCAST_BULL / BROADCAST_BEAR（传播）

### 3.3 从“Influence Graph”升级到“Influence-to-Trade”
当前已具备 influence graph 基础，但下一阶段要做到：

- 某条影响边会改变别的 clone 的交易概率
- 影响边会改变参与者的 confidence、entry threshold、exit threshold
- 影响边不只是可视化，而要进入 simulation 核心规则

### 3.4 从“状态图”升级到“市场状态机 + 净买力近似”
当前已有市场状态，但下一阶段要增强：

- 不同家族资金的进入 / 退出
- 净买力变化
- 关键 lead symbol / follow symbol 强度
- breadth 扩散程度
- crowding / fragility / invalidation 的更真实触发

### 3.5 从“页面可用”升级到“单事件演化墙”
当前 `event-sandbox` 可用，但终局要更像单事件舞台：
- 左侧事件图谱
- 中间参与者互动舞台
- 右侧 Belief / Scenario / MarketState
- 底部轮次回放
- 点击参与者、点击边、点击剧本均可深度追问

## 4. 下一阶段系统总流程

```text
事件输入
→ 事件结构化
→ Event Graph 构建
→ Market Regime 判定
→ Population Template 选择
→ Population Roster 生成
→ Clone 级参与者初始化
→ Clone 级交易 / 传播 / 退出
→ Influence-to-Trade 传播
→ Belief Graph 动态更新
→ Market State 迁移
→ Scenario 动态修正
→ Replay / Report / Validation
→ 真实事件包回填校准
```

## 5. 下一阶段核心模块

### 5.1 Event Graph Layer

#### 目标
把一条消息先变成可驱动后续行为的事件图谱。

#### 节点
- Event
- Commodity
- Sector
- Symbol
- ConfirmationSignal
- InvalidationSignal
- RiskFactor

#### 边
- impacts
- propagates_to
- confirms
- invalidates
- linked_to

#### 输出
- 结构化事件对象
- 图谱化事件上下文

---

### 5.2 Population Template Layer

#### 目标
定义当前市场环境下，各家族的：
- population_share
- capital_share
- influence_share
- default_clone_count
- style_distribution

#### 输出
- 一个明确的市场结构模板

---

### 5.3 Population Generator Layer

#### 目标
从模板生成具体的 Clone Roster。

#### 输入
- event_type
- market_regime
- template_id
- base_population_size

#### 输出
- `PopulationRoster`

#### 每个 clone 必须有
- clone_id
- participant_family
- style_variant
- capital_bucket
- max_event_exposure
- max_symbol_exposure
- reaction_latency
- entry/add/reduce/exit thresholds
- influence_weight
- current_position
- current_cash
- current_state

---

### 5.4 Capital Model

#### 目标
让系统不只是“谁看多”，而是“谁能动多少钱”。

#### 必须支持
- 资金范围桶（而不是精确金额）
- 单事件最大暴露
- 单标的最大暴露
- 每轮最大加仓比例
- 风险预算收缩规则
- post-loss shrink / cooldown 等行为规则（后续增强）

#### 本阶段输出
- clone 级 `capital_bucket`
- clone 级 `cash_available`
- clone 级 `position_limit`

---

### 5.5 Execution Clock

#### 目标
让 simulation 从抽象 round 变成更贴近真实市场的时间窗口。

#### 第一版建议窗口
1. pre_open
2. open_5m
3. morning_30m
4. midday_reprice
5. afternoon_follow
6. close_positioning

#### 作用
不同家族在不同时钟窗口反应不同：
- 游资更偏早盘
- 机构更偏中后段确认
- 散户更偏跟随与午后扩散
- 风控可在多个窗口触发

---

### 5.6 Execution Action Protocol V2

#### 第一版动作集合
- WATCH
- VALIDATE
- INIT_BUY
- ADD_BUY
- REDUCE
- EXIT
- BROADCAST_BULL
- BROADCAST_BEAR

#### 每个动作必须记录
- round_no
- execution_window
- order_side
- order_style
- order_size_pct
- order_value_range
- position_before
- position_after
- cash_before
- cash_after
- influenced_by
- reason_codes

---

### 5.7 Influence-to-Trade Conversion

#### 目标
让“影响图”真正改变交易行为。

#### 示例规则
- 媒体 `BROADCAST_BULL` 强度高 → 散户快钱 clone 的 `INIT_BUY / FOLLOW` 概率提升
- 产业链确认增强 → 机构 clone 的 `INIT_BUY / ADD_BUY` 概率提升
- 风控报警增强 → 量化与跟风 clone 的 `REDUCE / EXIT` 概率提升
- 龙头 lead 成功 + breadth 扩大 → Follow 类 clone 入场概率提升

#### 输出
- 影响边不仅可视化
- 还会改变参与者状态和交易概率

---

### 5.8 Market State Engine V2

#### 状态集合
- Dormant
- Ignition
- Propagating
- Crowded
- Fragile
- Invalidated

#### 下一阶段增强点
加入以下驱动量：
- 家族资金净流入近似
- clone lead / follow / exit 行为占比
- breadth 扩散近似
- confirmation vs invalidation 计数
- crowding 与 fragility 的联动规则

---

### 5.9 Replay / Report / Validation V2

#### Replay 必须看到
- 哪些家族被激活
- 各家族分身数量
- 哪些 clone 在什么窗口交易
- 哪些边触发了交易变化
- 市场状态如何迁移

#### Report 必须新增
- Population Summary
- Capital Summary
- Key Action Chain
- Success Chain / Failure Chain / Fragility Chain
- Participant Reliability
- Dominant Scenario Explanation

#### Validation 必须新增
- family-level hit rate
- clone-level reliability
- failure taxonomy
- market regime 下的表现统计

## 7. 本阶段开发顺序（建议 5 个 Sprint）

### Sprint 1：Population 基础层
目标：
- 引入 `ParticipantPopulationTemplate`
- 引入 `PopulationRoster`
- 实现 `PopulationGenerator`
- 页面上先能看见 family 与 clone

#### 验收
- 模板 A 可运行
- 24 个 clone roster 可生成
- 接上现有 `/participants/prepare`

---

### Sprint 2：Capital & Execution 基础层
目标：
- 引入 capital buckets
- 引入 execution clock
- 引入 `INIT_BUY / ADD_BUY / REDUCE / EXIT`
- clone 拥有真实资金范围和仓位变化

#### 验收
- 每个 clone 在 replay 中可以看到资金范围和买卖动作
- 日志里有 position/cash 前后变化

---

### Sprint 3：Influence-to-Trade 层
目标：
- influence edge 不只是显示
- 开始真正改变别的 clone 的交易行为

#### 验收
- 至少 4 类核心边可作用于交易概率
- replay 可显示“为什么这个 clone 在这一轮改变了行为”

---

### Sprint 4：Market State & Scenario 动态增强
目标：
- 市场状态迁移由 clone 行为驱动
- 三剧本按轮次修正
- 增加 key action chain / success chain / failure chain

#### 验收
- 同一事件至少能跑出 3~5 轮不同走向
- report 中可解释地说明为什么最终变成某个状态

---

### Sprint 5：Validation & 成品化收口
目标：
- 固定代表性真实事件包
- 增加 reliability scoreboard
- 增加 failure taxonomy
- 强化 event-sandbox 成品感
- 修 formal flow / roadmap 治理

#### 验收
- 至少 30~50 条真实事件可回放
- validation 页面能做 family / clone 层统计
- README / event-sandbox / live demo 表达与真实能力对齐

## 10. 本阶段最终验收标准

如果本阶段完成，Versefina 不再只是：

- 8 类参与者观点推演系统

而会升级为：

**一个带人口分布、资金分布、影响分布、交易时钟、买卖动作、市场状态演化、replay / report / validation 闭环的金融事件交易沙盘。**

必须达到的结果：

### 结果 1
用户看到的不再只是“谁看多看空”，而是：
- 哪些群体先被激活
- 哪些群体在哪个窗口买入
- 买入多少范围
- 谁带动了谁

### 结果 2
系统能更贴近真实世界地表达：
- 散户多但资金小
- 机构少但资金大
- 游资点火强
- 媒体放大叙事
- 风控压制扩散

### 结果 3
Replay 不再只是轮次列表，而是一段可解释的市场演化过程。

### 结果 4
Validation 不再只是“这个剧本对没对”，而能回答：
- 哪类家族更可靠
- 哪类人口结构更容易造成拥挤
- 哪类链条最容易失败

---

## 11. 一句话总结

**下一阶段的 Versefina，不是简单把 8 个 Agent 再做复杂一点，而是要正式升级成“人口级金融参与者交易沙盘”：让不同群体带着不同数量、不同资金、不同节奏、不同影响能力，围绕同一条事件在一个缩小版金融世界里进行买卖、传播、确认、压制与撤退。**
