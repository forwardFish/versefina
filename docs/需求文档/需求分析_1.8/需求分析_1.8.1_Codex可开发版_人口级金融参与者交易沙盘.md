## 2. 当前 Versefina 已完成状态（作为开发起点）

根据当前对照总结，Versefina 已有：

- 事件输入与结构化
- 参与者准备层
- Belief Graph / 场景推演 / 市场状态
- 轻量 simulation 与逐轮快照
- replay / report / validation
- 产品页 `event-sandbox`
- Swagger 手工验证面
- agentsystem 审计面
- `finahunt -> versefina` 真实事件导入
- 事件来源 lineage 展示

### 2.1 当前系统级能力判断

当前系统已经不是 PPT，也不是纯文档。

它已经处于：

**L3.5 ~ L4：可运行、可展示、可验证的事件级动态沙盘原型**

### 2.2 当前仍然缺什么

当前离你真正想要的“更像真实金融世界”的终局，还差：

1. 参与者家族 → 人口级 clone
2. 资金体量与仓位约束
3. 交易动作（买、加、减、退）正式化
4. 影响边真正改变交易行为
5. 市场状态与资金行为更强联动
6. 单事件演化墙成品感
7. validation / reliability / failure taxonomy 的强化
8. 治理与 authoritative roadmap 收口

## 3. 下一阶段唯一目标

### 3.1 一句话目标

把 Versefina 从：

**“8 类参与者家族级事件沙盘”**

升级为：

**“人口级金融参与者交易沙盘”**

### 3.2 这个阶段的核心变化

从现在开始：

- 8 个 Agent 不再等于 8 个单体角色
- 8 个 Agent family 变成 8 个参与者家族
- 每个家族按市场结构生成多个 clone
- 每个 clone 具有：
  - 资金体量范围
  - 仓位约束
  - 时钟偏好
  - 触发阈值
  - 跟风/引导/退出规则
- simulation 不再只是“观点动作”
- simulation 升级为“交易动作 + 传播动作 + 市场状态迁移”

---

## 4. 本阶段完成后，你最终会得到什么

一条消息进来后，系统将做到：

1. 先构建事件图谱
2. 根据市场结构模板生成参与者人口分布
3. 为每个 clone 分配风格、资金范围、仓位约束、执行窗口
4. 不同 clone 在不同窗口做：
   - WATCH
   - VALIDATE
   - INIT_BUY
   - ADD_BUY
   - REDUCE
   - EXIT
   - BROADCAST_BULL
   - BROADCAST_BEAR
5. influence graph 不只展示，而是驱动交易概率
6. market state 根据资金行为、扩散情况、证伪链条持续变化
7. replay 能展示谁买了、什么时候买、买多少范围、谁影响了谁
8. report 能展示成功链、失败链、脆弱链
9. validation 能统计家族级和 clone 级可靠性

## 5. 统一设计原则（Codex 必须遵守）

### 原则 1：不推翻现有事件主链
必须在当前 Versefina 主链上增强，不得重写成另一套系统。

### 原则 2：先做“分布级真实”，不做“逐个真实玩家复刻”
目标是贴近真实世界结构，而不是 1:1 模拟某个真实基金或席位。

### 原则 3：人数占比、资金占比、影响占比必须分开建模
不能把 “人多” 等价成 “钱多” 或 “影响大”。

### 原则 4：先支持一类事件
本阶段默认仍然只对：
- 涨价链
- 供给冲击
- 产业链传导  
做最强支持。

### 原则 5：先做可解释交易沙盘，不做自动实盘
所有交易动作都仅用于 simulation 和 replay，不涉及真实下单。

## 9. API 改造要求

## 9.1 参与者准备层增强接口

### `POST /events/{event_id}/participants/prepare-v2`
功能：
- 选择 population template
- 生成 clone roster
- 返回 family 与 clone 摘要

### `GET /events/{event_id}/participants/roster`
功能：
- 获取当前事件的家族与 clone 列表
- 包括资金范围、状态、权重

---

## 9.2 模拟层增强接口

### `POST /events/{event_id}/simulate-v2`
功能：
- 按 execution clock 跑一整轮或多轮 simulation
- 产出 clone 级动作与更新后的 market state

### `GET /events/{event_id}/simulation/rounds/{round_no}/actions-v2`
功能：
- 返回本轮 clone 级动作

### `GET /events/{event_id}/simulation/rounds/{round_no}/population-state`
功能：
- 返回本轮所有 clone 当前状态摘要

---

## 9.3 报告与验证接口增强

### `GET /events/{event_id}/report/population-summary`
功能：
- 返回 family 分布、capital 分布、影响分布、关键 clone

### `GET /events/{event_id}/validation/family-scoreboard`
功能：
- 返回 family 级命中率

### `GET /events/{event_id}/validation/clone-scoreboard`
功能：
- 返回 clone 级可靠性

## 10. 页面改造要求（Codex 要直接做页面，不只是后端）

## 10.1 event-sandbox 页面升级方向

### 现在页面的问题
- 已经可用
- 但更像“结果面板”
- 不够像“一个市场在动”

### 升级目标
把它变成：

**单事件人口级金融参与者交易沙盘**

---

## 10.2 页面必须新增的 4 个区块

### A. Population Summary Panel
显示：
- 8 个 family
- 每个 family clone 数量
- 每个 family 资金占比
- 每个 family 影响占比

### B. Clone Activity Stage
显示：
- 当前 round / 当前 execution window
- 当前正在行动的 clone
- clone 动作卡片
- 买 / 加 / 减 / 退 / 广播

### C. Influence to Trade Graph
显示：
- source family / source clone
- target family / target clone
- 影响类型
- 影响强度
- 这条边如何改变了对方买卖行为

### D. Capital & Position Snapshot
显示：
- 各 family 当前净资金流向近似
- 当前 lead symbol / follow symbol
- 当前持仓聚集在哪些方向

---

## 10.3 点击交互要求

点击某个 clone，必须弹出：
- 家族
- 风格
- 资金范围
- 当前仓位范围
- 触发阈值
- 本轮动作
- 被谁影响
- 影响了谁

点击某条 influence edge，必须弹出：
- 为什么这条边被激活
- 它改变了哪个 clone 的什么行为
- 是增加 entry / add / follow 还是增加 reduce / exit

## 11. 分阶段开发顺序（这部分是给 Codex 最重要的）

> 不要求一次性做完，必须分阶段。

---

## Phase 1：Population 基础层（必须先做）

### 目标
让系统先从“8 个 family”升级成“8 个 family + 多个 clone”。

### 要做的事情
1. 新增 `ParticipantFamily` 数据对象
2. 新增 `ParticipantPopulationTemplate`
3. 新增 `PopulationGenerator`
4. 新增 `population_clones` 表
5. 改造 `participants/prepare` 为 `prepare-v2`
6. 页面增加 Population Summary Panel

### 验收标准
- 一个事件能生成 24 个左右 clone
- 不同 family clone 数量不同
- 页面能看到 family 与 clone 的数量分布
- 页面能看到 population / capital / influence 的摘要

---

## Phase 2：Capital & Execution 基础层

### 目标
让 clone 不只是“有名字”，而是“有钱、有仓位、有窗口”。

### 要做的事情
1. 新增 capital buckets
2. 新增 execution clock
3. 新增 `ExecutionActionV2`
4. 日志记录买入 / 加仓 / 减仓 / 退出
5. 页面增加 clone 资金和动作展示

### 验收标准
- replay 中能看到 clone 级买卖动作
- 每个动作包含金额范围 / 仓位前后变化
- event-sandbox 可以切换 execution window

---

## Phase 3：Influence-to-Trade 转换层

### 目标
让 influence graph 真正改变交易行为。

### 要做的事情
1. 新增 `InfluenceToTradeRule`
2. 把已有 influence graph 规则接入 simulation
3. 让 family / clone 的行为概率会因影响边变化
4. 页面上可查看“这条边如何导致交易行为改变”

### 验收标准
- 至少 4 种核心规则生效
- replay 中能看到“某个 clone 为何因为某条边而改变动作”

---

## Phase 4：Market State & Scenario 强化

### 目标
让市场状态更像真实金融世界，而不只是抽象 round 状态。

### 要做的事情
1. `MarketStateEngineV2`
2. 引入净买力近似
3. 引入 breadth 扩散近似
4. 三剧本按轮次修正
5. 增加 success / failure / fragility chains

### 验收标准
- 同一事件可产生 3~5 轮不同走向
- 能解释为什么状态从 Ignition 进入 Propagating / Crowded / Invalidated
- 报告能输出关键行动链

---

## Phase 5：Validation & 治理收口

### 目标
把“能跑”升级成“可验证、可复盘、可治理”。

### 要做的事情
1. 固定真实事件包 30~50 条
2. 做 family reliability scoreboard
3. 做 clone reliability scoreboard
4. 做 failure taxonomy
5. roadmap / registry / acceptance 对齐
6. README 与 demo 表达对齐当前真实能力

### 验收标准
- validation 页面能看 family / clone 级可靠性
- 能统计哪些 family 更常 lead 成功，哪些更常误导
- formal flow 绿色收口

---

本阶段的唯一优先级是：

**在现有 Versefina 主链上，升级成人口级金融参与者交易沙盘。**
