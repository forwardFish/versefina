# 金融反身性引擎 MVP 需求文档 v1.0

## 1. 项目名称

**金融反身性引擎 MVP**  
副标题：**基于 Agent 社会 + Belief Graph + 世界模型的事件推演系统**

---

## 2. 项目背景

传统金融 AI 产品大多停留在两类：

一类是资讯总结，核心是“发生了什么”。  
另一类是单模型判断，核心是“这可能利好/利空什么”。

它们的共同问题是：

- 无法把“市场参与者的预期”作为一等变量
- 无法表达不同参与者之间的共识、分歧、拥挤和证伪脆弱性
- 无法把“预期如何影响资金行为，资金行为又如何反过来修正预期”纳入闭环

本项目希望构建一个最小可行的**金融反身性引擎**：

不是直接预测价格，  
而是围绕事件构建：

**事件 → Agent 分歧 → Belief Graph → 资金状态迁移 → 三剧本 → 后验验证**

---

## 3. 项目目标

### 3.1 MVP 核心目标

验证以下命题是否成立：

**在特定事件类型中，把 Agent 社会里的共识/分歧作为输入，能比传统资讯总结更早、更稳地给出可验证的决策剧本。**

### 3.2 业务目标

在 MVP 阶段，产品要完成三件事：

1. 输入一条金融事件，自动生成 8 类 Agent 的立场与分歧
2. 将这些立场结构化为 Belief Graph，并驱动三剧本推演
3. 在 T+1 / T+3 对剧本做后验验证，形成可复盘账本

### 3.3 用户目标

让用户获得的价值不是“更长的分析”，而是：

- 更早知道哪条逻辑可能发酵
- 更清楚看到当前市场共识和关键分歧
- 更明确知道明天该看什么信号
- 更快识别一条逻辑何时失效

---

## 4. 非目标

MVP 阶段不做以下内容：

- 不做自动交易
- 不接券商实盘
- 不做分钟级高频撮合
- 不做全市场全资产覆盖
- 不做开放 Agent 市场
- 不追求“预测涨跌准确率”作为唯一指标
- 不做复杂多用户协作权限体系

---

## 5. 目标用户

### 5.1 核心用户

- A 股短线/事件驱动交易者
- 高频看资讯的研究员
- 财经内容创作者
- 中小投研团队

### 5.2 典型使用场景

- 盘后看到一条政策消息，想判断明天如何发酵
- 看到上游原料涨价，想推演产业链扩散路径
- 某公司或行业出现负面消息，想推演利空扩散范围
- 想做盘前剧本，而不是看一堆碎片资讯

## 6. MVP 事件范围

第一版只做以下三类事件中的一种，建议优先只选一类上线：

### 6.1 政策催化

例如：

- 产业扶持政策
- 监管放松/收紧
- 财政、货币、行业政策变化

### 6.2 涨价链

例如：

- 上游原料涨价
- 供给受限
- 涨价沿产业链传导

### 6.3 利空扩散

例如：

- 公司爆雷
- 行业监管处罚
- 财务/舆情负面传播

---

## 7. 产品核心逻辑

### 7.1 旧范式

新闻 → 总结 → 利好/利空判断 → 个股/板块

### 7.2 新范式

新闻/公告/事件  
→ 8 类 Agent 形成观点  
→ 抽取共识/分歧结构  
→ 构建 Belief Graph  
→ 模拟资金状态迁移  
→ 输出三剧本  
→ T+1/T+3 验证与复盘

---

## 8. 核心概念定义

### 8.1 Agent 社会层

一组具有不同角色、仓位倾向和反应规则的金融参与者代理。

### 8.2 Belief Graph

围绕某一事件形成的“市场信念图谱”，用于表示：

- 谁支持
- 谁反对
- 谁高置信
- 谁观望
- 哪些观点会扩散
- 哪些观点容易证伪

### 8.3 资金状态迁移

不是直接预测价格，而是模拟不同资金风格在事件影响下如何：

- 关注
- 建立观察仓
- 加强仓位
- 跟随扩散
- 风险撤退
- 对冲
- 放弃

### 8.4 三剧本

- 基准剧本
- 乐观剧本
- 风险剧本

---

## 9. MVP 功能范围

### 9.1 Event Ingestion

输入一条事件，来源包括：

- 新闻
- 公告
- 快讯
- 用户手动粘贴文本

输出：

- 标题
- 正文
- 时间
- 来源
- 初步事件类型

### 9.2 Event Structuring

将原始事件结构化为：

- event_id
- event_type
- event_time
- entities
- sectors
- commodities
- affected_symbols
- causal_chain
- monitor_signals
- invalidation_conditions

### 9.3 Belief Engine

调用 8 类 Agent 对同一事件形成立场、置信度与预期。

输出：

- stance
- confidence
- time_horizon
- expected_impact
- evidence
- dissent_points
- trigger_conditions
- invalidation_conditions

### 9.4 Belief Graph

把 Agent 输出结构化，生成“信念图谱”。

输出：

- 共识强度
- 分歧宽度
- 关键支持方
- 关键反对方
- 关键少数
- 拥挤度
- 脆弱点

### 9.5 Scenario Engine

基于事件结构化结果 + Belief Graph，推演三剧本。

输出：

- 基准剧本
- 乐观剧本
- 风险剧本
- 关键观察点
- 资金状态迁移路径
- 失效条件

### 9.6 Report Card

面向用户展示一张事件推演卡。

页面展示：

- 事件摘要
- 8 个 Agent 立场
- Belief Graph 摘要
- 三剧本
- 关键观察点
- 失效条件
- 哪些 Agent 不同意当前主流结论

### 9.7 Outcome Tracker

记录 T+1 / T+3 实际结果。

包括：

- 板块表现
- 个股表现
- 情绪扩散情况
- 哪个剧本更接近现实
- 关键失误点
- Agent 表现评分

## 10. 8 个 Agent 定义

### 10.1 政策解读 Agent

职责：

- 判断政策方向、力度、覆盖范围、执行概率

输入：

- 政策文本/新闻

输出：

- 政策利好/利空方向
- 影响行业与板块
- 政策落地节奏判断

仓位倾向：

- 中低频，偏确认后参与

### 10.2 产业链 Agent

职责：

- 判断事件如何沿产业链上下游扩散

输出：

- 上游/中游/下游受益或受损路径
- 受益先后顺序

仓位倾向：

- 偏逻辑链路驱动

### 10.3 情绪扩散 Agent

职责：

- 评估事件是否适合题材化、情绪化传播

输出：

- 传播速度
- 是否具备短期情绪催化能力
- 是否容易形成市场热点

仓位倾向：

- 偏快节奏扩散

### 10.4 游资风格 Agent

职责：

- 评估事件是否适合短线资金博弈

输出：

- 是否有板块合力
- 是否适合做龙头/跟风/补涨

仓位倾向：

- 偏追涨和情绪合力

### 10.5 公募/机构风格 Agent

职责：

- 判断中线逻辑和机构资金是否会认可

输出：

- 是否具备基本面支撑
- 是否适合中线配置

仓位倾向：

- 偏确认后配置

### 10.6 量化/统计套利 Agent

职责：

- 判断当前事件是否已有 price-in，是否存在统计拥挤

输出：

- 情绪与价格是否过热
- 是否偏离历史同类事件

仓位倾向：

- 偏纪律和统计优势

### 10.7 风控 Agent

职责：

- 专门找反例、找脆弱点、找失败路径

输出：

- 当前逻辑最大风险
- 哪些确认信号不足
- 哪些变量一旦失效就要撤退

仓位倾向：

- 偏防守

### 10.8 复盘 Agent

职责：

- 对照历史同类事件、生成经验规则、归纳有效模式

输出：

- 历史相似事件
- 常见成功模式
- 常见失败模式

仓位倾向：

- 不直接交易，负责经验回灌

## 12. 三剧本生成逻辑

### 12.1 输入

- Event Structuring 输出
- Belief Graph 输出
- 当前行情状态
- 历史相似事件摘要

### 12.2 输出

#### 基准剧本

当前最可能发生的演化路径。

字段：

- narrative
- likely_path
- first_movers
- secondary_movers
- required_confirmation
- expected_duration
- risk_notes

#### 乐观剧本

如果关键确认信号成立，逻辑被强化后的演化路径。

字段：

- trigger_signals
- strengthening_factors
- expansion_path
- sector_rotation_path
- potential_leaders

#### 风险剧本

如果逻辑失效、共识崩塌、风控触发后的路径。

字段：

- failure_signals
- fragile_points
- likely_breakdown_path
- likely_first_casualties
- exit_hints

### 12.3 剧本生成规则

#### 基准剧本生成规则

优先使用：

- authority_weighted_consensus 较高
- dissent_width 中等
- crowding_score 未过高
- 风控未强烈反对

#### 乐观剧本生成规则

要求：

- 情绪扩散 Agent、游资 Agent、产业链 Agent 协同偏多
- propagation_speed 高
- 关键确认信号可观察

#### 风险剧本生成规则

要求：

- 风控 Agent 显著反对
- falsification_fragility 高
- 拥挤度高但确认不足
- 机构与情绪分歧大

## 13. 资金状态迁移模型

### 13.1 状态集合

- ATTENTION
- WATCH_POSITION
- CONVICTION_ADD
- TREND_FOLLOW
- RISK_REDUCE
- HEDGE
- ABANDON

### 13.2 状态转移规则示例

#### 规则 1

共识强度快速提升且传播速度高  
→ ATTENTION → WATCH_POSITION / TREND_FOLLOW

#### 规则 2

分歧扩大但价格未跌  
→ WATCH_POSITION → CONVICTION_ADD

#### 规则 3

权威 Agent 转空  
→ CONVICTION_ADD / TREND_FOLLOW → RISK_REDUCE

#### 规则 4

关键确认信号未兑现  
→ WATCH_POSITION → ABANDON

#### 规则 5

拥挤度过高且风控增强  
→ TREND_FOLLOW → HEDGE / RISK_REDUCE

## 14. 验证指标

### 14.1 基线对照

#### 基线 1

人工资讯总结，例如：

- 盘前快讯摘要
- 普通研报摘要

#### 基线 2

单 Agent 分析：

- 一个大模型直接看新闻给判断

#### 基线 3

纯行情因子：

- 只看价格/成交量/热度，不看 Agent 社会

### 14.2 核心验证指标

#### 1. 提前量

定义：系统比基线更早多久给出正确剧本框架

指标：

- 平均提前分钟/小时
- T+1 前关键信号发现率

#### 2. 稳定性

定义：同类事件下输出是否更一致、更少乱跳

指标：

- 同类事件剧本一致性
- 二次运行偏差率

#### 3. 校准度

定义：系统给高概率的剧本，后来是否更常发生

指标：

- probability bucket accuracy
- calibration error

#### 4. 失效识别速度

定义：逻辑崩掉时系统是否更快提示撤退

指标：

- 平均失效识别时间
- 相对基线提前量

## 19. 验收标准

MVP 验收以以下标准为准：

1. 至少支持 1 类事件类型稳定运行
2. 至少支持 8 类 Agent 输出结构化立场
3. 至少 50 条真实事件可形成完整样本
4. 每条样本都能输出三剧本
5. 至少 30 条样本具备 T+1/T+3 后验验证
6. 在“提前量 / 稳定性 / 校准度 / 失效识别速度”中，至少 2 项优于基线

## 20. 关键风险

### 20.1 Agent 输出空泛

解决：

- 强结构化输出
- 限制自由发挥
- 要求 evidence 和 trigger condition

### 20.2 Belief Graph 只是形式化包装

解决：

- 所有聚合指标必须真正参与 Scenario Engine
- 不允许只展示不计算

### 20.3 剧本很像研报空话

解决：

- 强制输出：
  - 关键观察点
  - 触发条件
  - 失效条件
  - 首动对象/后动对象

### 20.4 验证口径混乱

解决：

- 先统一事件类型
- 先统一 T+1/T+3 口径
- 先统一 dominant_scenario 判定规则

---

## 21. 一句话总结

**这不是一个“更会总结新闻”的金融 AI。**  
它是一个最小可行的 **金融反身性引擎**：

通过 8 类 Agent 形成共识/分歧，  
将其结构化为 Belief Graph，  
再驱动资金状态迁移和三剧本推演，  
最终通过后验验证形成可复盘、可评分、可迭代的决策系统。
