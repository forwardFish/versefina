## Goal
Turn the dynamic event sandbox into product-facing pages so one incoming message can be submitted, replayed, inspected round by round, and validated through a human-readable interface.

Sprint roadmap_1_7_sprint_5_event_sandbox_pages should deliver the following story goals in one coordinated cycle:
- Create /event-sandbox so one incoming message can be submitted, structured, prepared, and sent into simulation from a product-facing page.
- Create /event-sandbox/[eventId] so the user can see the event card, participant roster, round timeline, influence network, and market-state changes in one place.
- Create /event-sandbox/[eventId]/replay so the user can replay each simulation round and watch how actions, influence, belief, and market state evolve.
- Create /event-sandbox/[eventId]/participants/[participantId] so the user can inspect one participant's actions, inputs, outputs, and influence path across rounds.
- Create /event-sandbox/[eventId]/validation so the user can see report, why, validation, outcome, and reliability together on one product-facing page.
